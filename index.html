Linux coredump机制

发表于 2020-08-08

基础概念

Linux信号处理机制的一种，收到对应信号，会产生核心存储文件，同时进程终止。core文件会把保存当时进程的内存信息，可以通过gdb进行调试。

默认行为core的信号列表

名称 信号值 描述
SIGQUIT 3 终端退出
SIGSEGV 11 无效的内存引用
SIGSYS 31 终端退出

core文件生成限制

  1. ulimit -a 或者 unlimit -c可以看到
  2. core file size代表 core文件大小：0代表 core无法生成
  3. core生成，受到 core file size 和file size两个限制
