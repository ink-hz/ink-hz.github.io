编译过程

预处理阶段

预处理器(cpp)根据字符#开头指令，对原始的c程序进行修改。

The C preprocessor is the macro preprocessor for the C, Objective-C and C++ computer programming languages. The preprocessor provides the ability for the inclusion of header files, macro expansions, conditional compilation, and line control.

In many C implementations, it is a separate program invoked by the compiler as the first part of translation.

The language of preprocessor directives is only weakly related to the grammar of C, and so is sometimes used to process other kinds of text files.

The C Preprocessor is not a part of the compiler, but is a separate step in the compilation process. In simple terms, a C Preprocessor is just a text substitution tool and it instructs the compiler to do required pre-processing before the actual compilation. We'll refer to the C Preprocessor as CPP.

All preprocessor commands begin with a hash symbol (#). It must be the first nonblank character, and for readability, a preprocessor directive should begin in the first column. The following section lists down all the important preprocessor directives

C preprocessor is used before compiler processing program. It pre scans source code to complete functions including header file, macro extension, conditional compilation, line control, etc. For the test code, the preprocessor only processes the header file. To get the output of the preprocessor, use the command gcc -E hello.c -o hello.i. because hello.i There are many contents in the file. Here is a description of the intercepted part.

编译阶段

The process of compiling is to convert the source code written in one programming language (here, C language in particular) into another programming language (here, assembly language in particular). Ahead we will hello.c The pretreatment is done hello.i File, now it's time to hello.i File into the assembly file hello.s. 汇编阶段

The process of assembly is to convert the source code written in assembly language into executable machine code. Usually, the object file contains at least two segments: code segment and data segment. The code segment contains the program instructions, which are generally readable, executable and not writable; the data segment is used to store all kinds of global variables or static data used in the program, which are generally readable, writable and executable. To get the output of the assembler, use the command gcc -o hello.o -c hello.c. Since hello.o is a binary file, it cannot be read. Here, we use the command objdump to disassemble the binary file to see its contents.

objdump -h hello.o displays the header information of each segment in hello.o

链接阶段

The process of linking is to make one or more target file link libraries (static library or dynamic library) generated by compiler or assembler into executable files. The static library will be linked with the object file generated by the assembly and packaged into the executable file [static link]. Its link to the function library is completed during the compilation period. The dynamic library will not be linked to the executable file when the program is compiled, but will be loaded when the program is running. If different applications call the same library, they only need one instance of the shared library in memory. Get the executable file linked by the linker and use the command gcc hello.o -o hello. If you want to see the library the executable depends on, you can use the command ldd hello.

ldd hello displays the libraries that hello depends on

【算法】常见排序算法分析

发表于 2020-10-14

冒泡排序

冒泡排序思想

  1. 循环遍历输入的数组（这层循环确定已排序区间和未排序区间）
  2. 内部循环遍历未排序的区间（这层进行冒泡比较和调换）
  3. 比较内部循环相邻两个数据是否满足比较条件（例如小于），不满足就进行交换，再接着向后遍历，一直到遍历完未排序区间
  4. 优化点：定义一个交换标记，有交换时置位true，未有交换时提前退出循环

冒泡排序算法分析

  1. 原地排序算法（只涉及原内存空间的排序，空间复杂度为O(1)）
  2. 相邻两个数据==时不交换，是稳定排序算法
  3. 时间复杂度：最好O(n) 一轮循环直接结束，无需进行交换；最坏O(n^2) 要进行排序的数据时逆序的；平均O(n^2)涉及两层n常量的循环

插入排序

插入排序思想

  1. 循环遍历输入的数组（这层循环确定已排序区间和未排序区间），拿到当前需要进行插入排序的值
  2. 内层循环递减遍历已排序区间的数组，与当前需要插入的值进行比较，与插入值相比满足比较条件的向后移，一直找到插入的位置break
  3. break之后的索引+1（因为偏移到了未满足条件的前一个值了）即是当前值正确插入的位置

插入排序分析

  1. 原地排序（只涉及原内存空间的排序，空间复杂度为O(1)）
  2. 当前需要插入的值与比较值==时即break跳出循环，这样可以保证两个相同的值在数组中的位置不变，是稳定排序算法
  3. 时间复杂度：最好O(n)，最坏O(n^2)，平均 O(n^2)

归并排序

归并排序思想

  1. 归并排序用的是分治思想，将1个大问题递归分解小问题，处理完小问题之后再进行合并，最终处理完整个问题
  2. 归并排序数组：将数组递归平分成2个小数组，针对前后两部分进行排序，排序完再讲前后两部分进行合并
  3. 归并分解公式 merge_sort(array, p, r) = merge_sort(array, p, (p+r)/2) + merge_sort(array, (p+r)/2 + 1, r)
  4. 递归终止条件：p >= r
  5. 归并排序的重点在于合并过程merge(array, p, q)：
    . 1）根据传入p,q，分别开辟前后两个子数组L和R
    . 2）分别存入array从p, (p+r)/2和(p+r)/2+1,r的数据
    . 3) 循环比较L和R的数据，将满足条件的值放在：array从p索引开始的位置，依次向后递增
    . 4) L和R有一个数组的数据被放空当前循环结束，然后将剩余未遍历完的数据放入array递增的后续位置

归并排序分析

  1. 非原地排序算法，merge的过程中开辟了新空间，空间复杂度为O(n)
  2. 当L和R子数组值进行比较时，值相同，填入L的值到array，这样可以保证两个相同的值在数组中的位置不变，是稳定排序算法
  3. 时间复杂度：归并排序不管数组输入的有序程序是如何，最好，最坏，平均时间复杂度都是O(nlogn)

快速排序

快排思想

  1. 快排利用的也是分治思想
  2. 从array的low到high之间选取任意位置一个数组作为分区点(pivot)
  3. 遍历array从low到high，将满足比较条件（例如小于）的数据置于左区间，不满足的置于右区间，中间填写pivot的数据
  4. 继续递归排序左区间和右区间
  5. 递归公式：quick_sort(array, low, high) = quick_sort(array, low, pivot-1) + quick_sort(array, pivot+1, high)
  6. 递归终止条件：low >= high

快排分析

  1. 原地排序算法
  2. partition过程中涉及交换操作，相同元素的先后顺序可能发生改变，非稳定的排序算法
  3. 时间复杂度：平均时间复杂度为O(logn)，极端情况下为O(n^2)
  4. 快排的关键在于partition过程，实现了原址重排
